# CVE database

`gisweep` ships a bundled CVE database at
`src/gisweep/data/cve_db.json`. The database is keyed by the canonical
NVD `vendor:product` string and is regenerated by the refresh script
that pulls live data from
`services.nvd.nist.gov/rest/json/cves/2.0`.

```bash
uv run python -m scripts.refresh_cve_db --rate-delay 7
```

## Schema

```jsonc
{
  "schema_version": 1,
  "generated_at": "2026-05-03T00:00:00+00:00",
  "source": "https://services.nvd.nist.gov/rest/json/cves/2.0",
  "products": {
    "esri:arcgis_server": [
      {
        "cve_id": "CVE-2020-35712",
        "summary": "Esri ArcGIS Server before 10.8 is vulnerable to SSRF…",
        "severity": "critical",
        "cvss_score": 9.8,
        "cvss_vector": "CVSS:3.1/AV:N/AC:L/PR:N/UI:N/S:U/C:H/I:H/A:H",
        "published": "2020-12-26T00:15:12.213",
        "references": ["https://nvd.nist.gov/vuln/detail/CVE-2020-35712"],
        "ranges": [
          { "introduced": null, "fixed": "10.8", "exact": null }
        ]
      }
    ]
  }
}
```

`ranges` covers both half-open `[introduced, fixed)` intervals and
specific-version `exact` matches that NVD uses when an advisory only
lists single versions in the affected CPE configurations.

## ArcGIS Server version normalisation

ArcGIS Server's `currentVersion` field uses a compact form (`10.91` for
10.9.1, `10.31` for 10.3.1). ARC-015 normalises that to the standard
semver form before the lookup so the CVE matching works against the
NVD-style `10.9.1` entries.
